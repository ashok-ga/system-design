    Module 2 – Notification Service (LLD)¶

    Overview¶

    The Notification Service handles all outgoing notifications—such as emails, SMS messages, and push notifications—triggered by various system events.

    Class Diagram¶

    Main Classes and Their Roles¶

    1. NotificationManager¶

    Central coordinator for all notification events. Determines channel(s) and dispatches to the appropriate sender class.

    class NotificationManager:
    def __init__(self, email_sender, sms_sender, push_notifier):
        self.email_sender = email_sender
        self.sms_sender = sms_sender
        self.push_notifier = push_notifier

    def send_notification(self, user, message, channels):
        """Send notification to user via specified channels."""
        results = {}
        if "email" in channels:
            results["email"] = self.email_sender.send_email(user.email, message)
        if "sms" in channels:
            results["sms"] = self.sms_sender.send_sms(user.phone, message)
        if "push" in channels:
            results["push"] = self.push_notifier.send_push(user.device_id, message)
        return results

    Handles all email-related sending logic (using SMTP or an external service).

    3. SmsSender¶

    Handles SMS sending (using a third-party gateway API).

    class SmsSender:
    def __init__(self, api_key):
        self.api_key = api_key

    def send_sms(self, phone, message):
        print(f"Sending SMS to {phone}: {message}")
        return True

    4. PushNotifier¶

    Handles push notifications (e.g., using Firebase Cloud Messaging).

    class PushNotifier:
    def __init__(self, fcm_key):
        self.fcm_key = fcm_key

    def send_push(self, device_id, message):
        print(f"Sending PUSH notification to device {device_id}: {message}")
        return True

    Example Usage¶

    # Example user object
class User:
    def __init__(self, email, phone, device_id):
        self.email = email
        self.phone = phone
        self.device_id = device_id

email_sender = EmailSender("smtp.mail.com", 587, "login", "password")
sms_sender = SmsSender("api-key-123")
push_notifier = PushNotifier("fcm-key-xyz")

notifier = NotificationManager(email_sender, sms_sender, push_notifier)

user = User("test@example.com", "+911234567890", "device123")
channels = ["email", "push"]
notifier.send_notification(user, "Welcome to System Design Hub!", channels)

    Sequence Flow¶

    1. An event triggers a notification.
    2. NotificationManager.send_notification() is called.
    3. It decides channels, then delegates sending to each sender class.
    4. Each sender class implements the sending logic (log, API call, etc).

    Error Handling and Extensibility¶

    • Each sender can log errors, retry failed deliveries, or raise exceptions.
    • New channels (e.g., WhatsAppSender) can be added by extending the manager.
